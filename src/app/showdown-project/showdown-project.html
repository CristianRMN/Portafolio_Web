Pokemon Showdown en Java

Proyecto realizado en java que simula una batalla Pokemon 3 vs 3

DESCRIPCIÓN

Este proyecto fue desarrollado como parte del fin del primer año del ciclo de DAM,
y consiste en una simulación de batallas Pokémon 3 vs 3 creada en Java utilizando
Swing. Aunque actualmente se utiliza JavaFX por su escalabilidad y su mejor separación
de responsabilidades siguiendo el patrón MVC, en aquel momento no se había impartido
dicho framework, por lo que el proyecto fue desarrollado completamente con Swing.
Esto representó un gran desafío, pero también una excelente oportunidad para profundizar
en los principios de la Programación Orientada a Objetos. El juego cuenta con un login
 donde el usuario puede elegir su icono de personaje y nombre de entrenador, seguido
de un sorteo tipo cara o cruz para decidir quién selecciona primero a sus tres Pokémon.
En esta versión se incluyen 12 Pokémon, aunque el diseño permite una futura ampliación
del catálogo. Se implementaron las mecánicas originales de combate de la saga, incluyendo
el algoritmo de daño utilizado por Nintendo. El sistema sigue una estructura de combate
por turnos donde el Pokémon más rápido ataca primero, con la posibilidad de cambiar de
Pokémon si uno es debilitado. El combate finaliza cuando los tres Pokémon de un jugador
son derrotados. La información de los Pokémon se gestiona mediante una base de datos
PostgreSQL, y se aplicaron principios de herencia en Java para la jerarquía de clases
que representan las diferentes especies y tipos de Pokémon.
