Backend de contenidos con Symfony

Prueba técnica realizada con Symfony para la gestión de contenidos CRM

DESCRIPCIÓN

Este proyecto fue desarrollado con Symfony como una aplicación de gestión de
contenidos “branded content”, similar a un CRM. El sistema se estructura en
cinco tablas principales: Proveedores, Medios, Contenidos, Tarifas y Enlaces.
Las relaciones entre entidades están cuidadosamente diseñadas para reflejar
la lógica de negocio: un Proveedor y un Medio se conectan a través de un
Contenido, que a su vez puede estar asociado a una Tarifa y a uno o varios Enlaces.
Esto define una relación de muchos a muchos entre proveedores y contenidos,
gestionada mediante la tabla de tarifas. Cada contenido debe tener asignado
un ID de proveedor y de medio, y solo puede existir si tiene una tarifa vigente.
Durante el desarrollo se implementaron 30 endpoints en tan solo dos días, con
autenticación JWT para la seguridad de la API. La prueba técnica se ejecutó
utilizando Docker y el entorno WSL para validar su correcto funcionamiento.
Se empleó MySQL como base de datos. Aunque la prueba incluía un frontend
en Angular, el enfoque principal fue el backend, siguiendo las indicaciones del
CEO de la empresa, quien priorizaba la lógica y estructura del servidor
como parte clave del proyecto.
